In this contrived example, a single component presents three button elements. These are all individually clickable, but from the user perspective, the buttons don't receive the focus. Instead, the focus gets placed on the overall component. This reinforces the idea that the buttons are acting as a single unit. This component also appears in the tab order, and can be reached via the Tab and Shift+Tab keys.

For an aggregate focus to make sense, the component must provide a way for the user to reach all its core functions with the keyboard, regardless of which element within the component actually has the real focus. In this example, pressing the "R", "G", or "B" keys while the component has the focus has the same effect of pressing the Red, Green, or Blue buttons, respectively.
